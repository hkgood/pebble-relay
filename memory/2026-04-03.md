### 关键决策：测试模式改造

**目标**：在 macOS 上完全复刻真实树莓派硬件流程
- SPACE 按下 = 按下按钮（开始录音）
- SPACE 松开 = 松开按钮（停止录音 → 转写 → LLM → TTS）
- 所有发送语音、获取回复的逻辑与真实部署完全一致

**实现方案**：
- `Recorder`（`record_audio.py`）：macOS 用 `sox rec` 录音（通过 `subprocess.Popen`），Linux/ARM 继续用 `arecord`
- `TTSPlayer`（`tts_openai.py`）：macOS 用 `afplay` 播放，Linux 用 `aplay`
- `_TestModePTT`（`main.py` 内嵌类）：pygame 子进程渲染按钮，通过 Unix socket 接收 SPACE 事件，主进程调用真实的 `start_recording()` / `_on_button_release()` 等方法

**依赖**：
- macOS：`pip3 install pygame sox`（`sox` = `brew install sox`）
- Linux：`pip3 install pygame`

**文件**：
- `test_input.py`：pygame 虚拟 PTT 窗口（新建）
- `record_audio.py`：增加 macOS `sox rec` 支持
- `tts_openai.py`：增加 macOS `afplay` 支持
- `main.py`：`_TestModePTT` 改造，复刻真实硬件流程
- `requirements.txt`：增加 `pygame`
